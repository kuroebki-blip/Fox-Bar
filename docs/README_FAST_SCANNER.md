# FO’X v15.13.3 / backend v9.4.4 — FAST SCANNER

## Файлы

- `frontend/candidate/index.html`
- `apps-script/stock-scanner/candidate/Code.gs`

## Что изменено

1. Документы и чеки больше не собираются в PDF до отправки в Gemini.
2. Для OCR создаются облегчённые JPEG-копии до 1800 px.
3. Gemini начинает распознавание сразу по изображениям.
4. Полный качественный PDF собирается параллельно и сохраняется отдельно для отправки в Telegram.
5. Кассовый отчёт распознаётся напрямую по JPEG, без PDF.
6. Команды статуса банкета и «Заказ отправлен» переведены на JSONP с проверяемым ответом backend.
7. В backend v9.4.4 итог терминала берётся строго из «Количество оплат → На сумму».
8. Повторное OCR не подгоняет терминальный итог под iiko.
9. Дата кассовой смены берётся только из длинного отчёта iiko.

## Что не меняется

- стоковые значения D/E/F;
- формулы B/G/J;
- Cloudinary;
- backend банкетов;
- `setupBanquetsBackend()` и любые setup-функции запускать не нужно.

## Установка

### Apps Script сканера/стока

1. Открыть `Code.gs`.
2. Заменить весь файл кодом `apps-script/stock-scanner/candidate/Code.gs`.
3. Сохранить.
4. Развернуть → Управление развертываниями → карандаш → Новая версия → Развернуть.
5. Проверить `?action=ping`: версия должна быть `v9.4.4 CASH REPORT SOURCE RULES FIX`.

### GitHub Pages

1. Заменить весь `index.html` файлом `frontend/candidate/index.html`.
2. Commit changes.
3. Открыть Mini App с `?v=15.13.3`.

## Тест

1. Один документ из 1 страницы.
2. Документ из 3 страниц.
3. Кассовый отчёт с двумя терминальными слипами.
4. Статус банкета: Актуальный → Пройден.
5. Новый банкет с фото → проверить лист `Банкеты_Резерв`.

## Важно

Банкетный резерв уже распознаёт фото напрямую из Cloudinary и не использует PDF-сканер. В этой версии исправлена надёжность команд статуса и заказа, но сопоставление новых позиций нужно проверить отдельным живым тестом.
