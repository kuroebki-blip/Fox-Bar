# FO’X — Stock Table Rules

## Основная задача

Таблица стока должна помогать быстро понять:
- сколько есть на Аллее;
- сколько есть в Заготовочном;
- сколько всего в остатке;
- сколько нужно заказать.

## Листы стока

Актуальные листы:
- Вино
- Крепкий алкоголь
- Пюре и сиропы
- Чай
- Пиво
- Прочее
- Посуда Бар

## Структура листов

```text
A Наименование
B ЗАКАЗ
C Сток
D Остаток Аллея
E Остаток Заготовочный
F Цена
G Сумма остатка
H Ед. изм.
I Примечание
```

## Формулы

Сумма остатка:
```text
G = D + E
```

Заказ:
```text
B = MAX(0, C - G)
```

Смысл:
- `C` — целевой сток;
- `G` — сколько всего есть;
- `B` — сколько надо заказать.

## Правила редактирования

1. Никогда не удалять формулы в B и G.
2. Никогда не сдвигать колонки без обновления Apps Script.
3. Не менять названия листов без обновления кода.
4. Новые позиции добавлять в конец соответствующего раздела.
5. Перед массовыми изменениями делать копию файла.
6. Для Google Sheets не импортировать XLSX поверх старой рабочей таблицы без теста.
7. Если создаётся новая XLSX-версия, обязательно проверить формулы после открытия в Google Sheets.

Текущие позиции:

## Apps Script стока

В коде обязательно должен быть лист `Пиво` в `stockSheets`.

Пример:

```js
stockSheets: [
  'Вино',
  'Крепкий алкоголь',
  'Пюре и сиропы',
  'Чай',
  'Пиво',
  'Прочее'
]
```

## Что проверять после правок

1. В каждом стоковом листе есть колонки A-I.
2. В G стоит сумма D+E.
3. В B стоит MAX(0,C-G).
4. Формулы протянуты на все товарные строки.
5. Нет `#REF!`.
6. Визуальный стиль не поплыл.
7. Apps Script меню появляется.
8. Функция заказа не падает.
