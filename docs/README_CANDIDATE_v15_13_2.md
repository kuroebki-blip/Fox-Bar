# FO’X v15.13.3 / Backend v9.4.4

## Что исправлено

- В предупреждении о расхождении терминалов больше не показывается домен GitHub Pages.
- Вместо браузерного `confirm()` используется собственное окно Mini App.
- Дата кассовой смены берётся из длинного отчёта iiko.
- Даты банковских слипов и конверта используются для контрольной сверки.
- Добавлена проверка полноты платёжных строк iiko через итоги фискального и нефискального разделов.
- Если строка `Наличка` пропущена, но итог нефискального раздела не сходится, запускается узкая повторная проверка iiko.
- Сохранены: быстрый JPEG pipeline, QR-проверка терминалов и повторная отправка отчёта.
- Старое фоновое создание PDF не может изменить новое задание после сброса сканера.
- Добавлены ранние ограничения: максимум 20 страниц и 12 МБ OCR-изображений.
- Итог терминального слипа и дата кассовой смены защищены правилами backend v9.4.4.

## Какие файлы меняются

1. `index.html`
2. Apps Script сканера/стока `Code.gs`

Стоковые значения, формулы и backend банкетов не меняются.

## Установка

### Apps Script

1. Заменить весь `Code.gs` файлом `apps-script/stock-scanner/candidate/Code.gs`.
2. Сохранить.
3. Развернуть → Управление развертываниями → карандаш.
4. Новая версия → Развернуть.
5. Никакие `setup...` не запускать.

### GitHub

1. Заменить весь `index.html` файлом `frontend/candidate/index.html`.
2. Commit changes.
3. Открыть Mini App с `?v=15.13.3`.

## Проверка

1. Кейс с минусом по терминалам: окно должно открыться без названия репозитория.
2. Проверить дату смены.
3. Проверить строку `Нал` на отчёте, где `Наличка` находится в нефискальном разделе.
4. Проверить QR и повторную отправку сообщения.
