## Состав

- `frontend/tatooine/index.html` — только интерфейс кассового отчёта;
- `frontend/tatooine/styles.css` — отдельные стили Tatooine;
- `frontend/tatooine/config.js` — публичная конфигурация и существующий Web App URL;
- `frontend/tatooine/app.js` — фотографии, OCR polling, проверка терминалов, редактирование и повторная отправка.

Приложение не содержит чек-листы, банкеты, документы, чеки и управление стоком.

## Отдельный Telegram-бот

Tatooine использует тот же Web App URL, но передаёт маршрут `venue=tatooine`. Backend проверяет `initData` токеном отдельного бота и отправляет результат через него. Задания FO’X и Tatooine разделены режимом задания, поэтому один бот не может прочитать или отправить отчёт другого.

В Script Properties нужно сохранить:

- `TATOOINE_TELEGRAM_BOT_TOKEN` — токен нового бота от BotFather;
- `TATOOINE_TELEGRAM_TARGET_CHAT_ID` — необязательно; без него используется текущий чат или личный чат пользователя;
- `TATOOINE_TELEGRAM_ALLOWED_USER_IDS` — необязательно; список Telegram ID через запятую, по умолчанию используются сохранённые admin IDs FO’X.

Токен не хранится во frontend, GitHub или Google Sheets.

## Шаблон сообщения Tatooine

Candidate v1.1.0 формирует сообщение по эталону Tatooine:

- заголовок `TATOOINE` и цитата `ПЕТРОВКА`;
- дата в коротком формате `ДД.ММ.ГГ`;
- общая выручка, безнал, наличные, Онлайн касса 2, EatAndSplit и Яндекс еда;
- нулевые дополнительные способы оплаты скрываются;
- `Расчётный счёт` и `Расчётный счёт 2` остаются отдельными значениями и выводятся независимо;
- расход, инкассация, неизменный размен;
- список предоплат и строка `Итого`.

22 июля 2026 `tatooineCashCaptureTelegramStyle()` успешно сохранил логотип из 8 символов и оформление всех 11 строк. Функция сохранила только идентификаторы оформления в Script Properties и не изменяла таблицы или изображения.

Frontend v1.1.1 копирует выбранные фотографии до очистки file input. Это устраняет потерю `FileList` в Telegram WebView после закрытия камеры или галереи; живой повторный тест ожидается.

Frontend v1.2.0 повторяет оригинальные пустые строки, эмодзи и выделения. Значения полей `Нал` и `Нал Фискал` складываются только при формировании сообщения и выводятся одной строкой `Нал`; распознанные поля в форме остаются раздельными.

## Сохранённые правила кассового отчёта

- итог слипа берётся из «Количество оплат → На сумму»;
- «Оплата QR» участвует в проверке;
- `Расчётный счёт` и `Расчётный счёт 2` остаются отдельными строками;
- дата смены берётся из длинного отчёта iiko;
- инкассация сохраняется в формате `значение (значение в скобках)`;
- исправленный отчёт можно отправить повторно;
- фотографии не сохраняются в Google Sheets и Google Drive.
