# Tatooine — кассовый отчёт

## Версия

- frontend: `v1.0.0 CASH REPORT ONLY`;
- backend: существующий FO’X scanner/stock Web App `v9.4.4`;
- статус: candidate, автоматические проверки пройдены, Telegram-тест и публикация не выполнены.

## Состав

- `frontend/tatooine/index.html` — только интерфейс кассового отчёта;
- `frontend/tatooine/styles.css` — отдельные стили Tatooine;
- `frontend/tatooine/config.js` — публичная конфигурация и существующий Web App URL;
- `frontend/tatooine/app.js` — фотографии, OCR polling, проверка терминалов, редактирование и повторная отправка.

Приложение не содержит чек-листы, банкеты, документы, чеки и управление стоком.

## Важное ограничение существующего обработчика

Backend проверяет Telegram `initData` токеном, который хранится в его Script Properties. Поэтому Tatooine должен запускаться из того же Telegram-бота, что и FO’X. Отправка идёт в `TELEGRAM_TARGET_CHAT_ID` этого обработчика.

Если Tatooine должен использовать другой бот или другой целевой чат, потребуется отдельный backend либо безопасная многоресторанная маршрутизация на стороне Apps Script. Frontend нельзя переключать на другой бот без изменения backend.

## Сохранённые правила кассового отчёта

- итог слипа берётся из «Количество оплат → На сумму»;
- «Оплата QR» участвует в проверке;
- `Расчётный счёт` и `Расчётный счёт 2` остаются отдельными строками;
- дата смены берётся из длинного отчёта iiko;
- инкассация сохраняется в формате `значение (значение в скобках)`;
- исправленный отчёт можно отправить повторно;
- фотографии не сохраняются в Google Sheets и Google Drive.

## Проверка перед публикацией

1. Открыть Tatooine из существующего Telegram-бота.
2. Проверить отчёт iiko и два терминальных слипа, один из них с QR.
3. Проверить расхождение терминалов и окно подтверждения.
4. Проверить текст: первая строка `Tatooine`, затем `РЕСТОРАН TATOOINE`.
5. Отправить отчёт в тестовый чат и повторно отправить исправленную версию.

## Откат

Вернуть предыдущий frontend-файл в репозитории Tatooine или отключить новую кнопку Mini App в Telegram. Существующий FO’X backend при публикации этого frontend не изменяется.
