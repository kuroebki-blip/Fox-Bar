# FO’X — контекст решений из ChatGPT

Этот файл заменяет перенос длинной истории чата. Codex должен опираться на исходники, этот документ и `AGENTS.md`.

## Архитектура

- Frontend: GitHub Pages, один большой `index.html`.
- Telegram Mini App Hub: чек-листы, банкеты, документы/чеки, кассовые отчёты.
- Банкеты: отдельный Apps Script + Google Sheets + Cloudinary.
- Сток/сканер/касса: отдельный Apps Script + основная Google Sheets.
- Gemini используется через Apps Script; API key не хранится во frontend.

## Банкетный модуль

- Фото грузится напрямую в Cloudinary.
- Google Sheets хранит URL.
- Администраторы определяются Telegram user ID.
- Статусы должны отображаться и сохраняться.
- В одной дате допускается несколько банкетов.
- Банкетный резерв агрегирует несколько банкетов одной позиции.
- «Заказ отправлен» снимает резерв, чтобы не заказывать повторно каждую неделю.

## Сканер

- Встроенная камера Android, HD, вспышка, ручная обрезка и поворот.
- Автоопределение границ отключено из-за скорости.
- Документы могут быть многостраничными.
- Отправка сообщения доступна даже для позиции, отсутствующей в стоке.

## Кассовый отчёт

- Общая выручка — из iiko без изменений.
- Безнал — из `Банковские карты`; сравнивается с суммой терминальных слипов.
- Терминальный итог — строго `Количество оплат → На сумму`.
- QR должен учитываться в валидации.
- Нал — `Наличка` в нефискальных типах.
- Нал Фискал — наличные в фискальных типах.
- `Банковские карты 2`, `Наличные 2`, Tapper, `Расчётный счёт`, `Расчётный счёт 2`, `Онлайн-Касса 2` — отдельные точные строки.
- Групповые итоги нельзя подставлять вместо платёжной строки.
- Инкассация берётся с конверта и выводится как `71 938 (71 940)`.
- Расход: одно поле + комментарий.
- Предоплаты вводятся вручную: дата, сумма, способ оплаты.
- Пустые платёжные строки остаются в сообщении.
- Отчёт можно повторно отправить после исправления.
- Telegram-оформление использует custom emoji, найденные через Script Properties.
- Начиная с candidate backend v9.4.4 итог терминального слипа не пересчитывается по компонентам, если распознан блок «Количество оплат → На сумму».
- Повторное OCR терминалов оценивается по внутренней согласованности слипа, а не по близости к iiko.
- Даты терминальных слипов и конверта не заменяют дату длинного отчёта iiko.
