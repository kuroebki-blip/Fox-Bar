# FO’X — Changelog

## Tatooine frontend v1.0.0 — CASH REPORT ONLY

Статус: candidate от 22 июля 2026; автоматические проверки пройдены, Telegram-тест и публикация ожидаются.

Что добавлено:

- отдельное приложение `Tatooine` только для кассового отчёта;
- модульная структура `index.html` + `styles.css` + `config.js` + `app.js`;
- использование существующего Apps Script обработчика FO’X v9.4.4;
- загрузка нескольких фото, изменение порядка, поворот и удаление;
- OCR-лимиты 20 страниц и 12 МБ до отправки;
- редактирование сумм, сверка терминалов, предоплаты и повторная отправка;
- сообщение Telegram с брендом `Tatooine`.

Как откатиться:

- вернуть предыдущий frontend в отдельном репозитории Tatooine или отключить кнопку Mini App;
- Apps Script откатывать не требуется, потому что backend не изменяется.

## Frontend v15.13.3 — SCANNER JOB SAFETY

Статус: production с 22 июля 2026; GitHub Pages опубликован, браузерный smoke-тест пройден, Android/Telegram-тест ожидается.

Что изменено:

- старое фоновое создание PDF больше не может записать URL в результат нового сканирования;
- PDF строится по снимку страниц конкретного задания;
- отменённая PDF-загрузка прекращает polling и не меняет глобальное состояние;
- изменение, удаление и перестановка страниц документа или кассового отчёта отменяют результат предыдущего задания;
- до OCR проверяется максимум 20 страниц;
- общий размер OCR-изображений ограничен 12 МБ до отправки в Apps Script.

Файлы:

- `frontend/candidate/index.html`;
- `tests/frontend/scanner_state.test.js`;
- документация состояния проекта.

Что проверить вручную:

- сбросить сканер во время подготовки PDF и сразу распознать другой документ;
- документ из 1 и 3 страниц на Android;
- кассовый отчёт с длинным iiko, конвертом и двумя терминальными слипами;
- подтверждение стока и отправку документа без внесения позиции в сток.

Как откатиться:

- отменить GitHub merge commit `10f766c11deabfe0403af1640e266e6dbc998a6f`;
- дождаться повторной сборки GitHub Pages с предыдущим `index.html`.

## Stock/scanner backend v9.4.4 — CASH REPORT SOURCE RULES FIX

Статус: production с 22 июля 2026; Apps Script version `16`, публичный `ping` вернул `v9.4.4`, живой Telegram-сценарий ожидается.

Что изменено:

- итог терминального слипа берётся строго из блока «Количество оплат → На сумму»;
- сумма «Оплата + Оплата QR» используется только при отсутствии итогового блока;
- повторное OCR терминалов больше не выбирается по близости к сумме iiko;
- дата кассового отчёта берётся только из длинного отчёта iiko;
- добавлены автоматические тесты правил кассового отчёта.

Файлы:

- `apps-script/stock-scanner/candidate/Code.gs`;
- `tests/cash/cash_report_logic.test.js`;
- документация состояния проекта.

Что проверить вручную:

- отчёт с совпадающими iiko и терминалами;
- расхождение терминалов −1 000 ₽ не исчезает после повторного OCR;
- терминальный итог с QR берётся из нижнего итогового блока;
- дата терминала или конверта не заменяет дату iiko;
- исправленный отчёт повторно отправляется в Telegram.

Как откатиться:

- переключить рабочее Apps Script-развёртывание с version `16` обратно на version `15`;
- проверить `?action=ping`, он должен снова вернуть `v9.4.3 CASH REPORT VALIDATION FIX`.
