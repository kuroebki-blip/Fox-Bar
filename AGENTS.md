# FO’X — обязательные правила для Codex

## Общий порядок

1. Сначала изучи `README.md`, `docs/PROJECT_STATE.md`, `docs/CHAT_CONTEXT.md` и затрагиваемые исходники.
2. До правки назови точные файлы, которые собираешься менять.
3. Делай минимальный diff. Не переписывай рабочую систему вслепую.
4. Не утверждай, что функция работает, пока нет теста пользователя или автоматического теста.
5. Каждый релиз получает версию и запись в changelog.
6. Перед изменениями создай ветку или отдельный коммит.

## Банкетная система

- Рабочая схема: Mini App → Cloudinary → URL → Apps Script → Google Sheets.
- Не возвращай загрузку фото через Apps Script/base64.
- Не запускай `setupBanquetsBackend()` без прямого подтверждения пользователя.
- Не меняй banquet backend, если задача только во frontend.
- Поддерживай несколько банкетов на одну дату.
- Одна позиция может участвовать в нескольких банкетах.
- Статусы: Актуальный / Пройден / Отменён.
- Кнопка «Заказ отправлен» снимает резерв, но не удаляет банкет.

## Сток

- Не удаляй формулы, значения и стили без отдельного подтверждения.
- Перед массовыми правками делай новую копию таблицы/файла.
- Базовая структура: A Наименование, B ЗАКАЗ, C Сток, D Остаток Аллея, E Остаток Заготовочный, F Цена, G Сумма остатка, H Ед. изм., I Примечание, J Банкеты.
- Базовая логика: G = D + E; B = MAX(0, C - G + J) для версии с банкетным резервом.
- Не сдвигай колонки и не переименовывай листы без обновления Apps Script и тестов.

## Сканер документов и кассовый отчёт

- Фото не должны проходить через Google Sheets.
- Ключи Gemini и Telegram хранятся только в Script Properties.
- Для терминальных слипов итог брать из блока «Количество оплат → На сумму».
- «Оплата QR» должна учитываться при проверке итогов.
- `Расчётный счёт` и `Расчётный счёт 2` — отдельные строки.
- Инкассацию брать с надписи на конверте и сохранять формат `значение (значение в скобках)`.
- Разрешена повторная отправка исправленного кассового отчёта.
- Отправка сообщения о документе/чеке разрешена, даже если позиция отсутствует в стоке.

## Секреты

Не коммить:

- `GEMINI_API_KEY`;
- `TELEGRAM_BOT_TOKEN`;
- Cloudinary API Secret;
- приватные ключи и пароли;
- экспорт Script Properties.

Публичные Web App URL и Cloudinary unsigned preset допустимы, но перед публикацией проверь, что в коде нет секретов.

## Проверка перед релизом

- нет заглушек `PASTE_...`;
- сохранены Telegram admin IDs;
- сохранены рабочие Web App URL;
- нет удаления setup-функциями;
- проверены банкеты без фото и с фото;
- проверены документы и кассовый отчёт;
- проверены формулы B/G/J;
- указан способ отката.
